The TCP/IP Networking Model:

The TCP/IP Networking Model has been in development for some time, replacing the antequated and unorganized proprietary protocols of the past. It provides a layered networking model and standard for different organizations to work with to allow for seamless communication between disparate systems.

Table 1-2 TCP/IP Architectural Model and Example Protocols:

TCP/IP Architectural Layer Example Protocols
Application HTTP, POP3, SMTP
Transport TCP, UDP
Internet IP
Link Ethernet, Point to Point (PPP), T1

TCP/IP Application Layer

Provides services to network applications. It does not define the application, but only the services it needs to access the network. As noted above, a popular application layer protocol is HTTP, which is used by web browsers.

TCP/IP Transport Layer

The transport layer includes a smaller number of protocols than other layers. The two most commonly used transport layer protocols are Transmission Control Protocol (TCP) and User Datagram Protocol (UDP). This layer provides services to the higher layer protocols.

TCP Error Recovery

TCP uses sequence numbers to transmit data, lets say bob sends larry four messages but larry only gets messages 1, 3, and 4. Larry’s computer will send a message to bobs to request the missing data in message (segment) 2.

Same layer and adjacent layer interactions

The example above where bob and larrys computers are communicating and one sends the other a request for a new TCP segment, this is a same layer interaction, because they are both using TCP (Transport Layer) for this type of interaction.

Adjacent layer interactions are typically on the same computer as the data either goes up or down the TCP/IP stack, so to speak. So, HTTP making TCP segments to send to another computer would be a good example of this.

TCP/IP Network Layer

The Network Layer provides a relatively small number of protocols as well. Most notably it provides Internet Protocol (IP). IP provides several features, most notably addressing and routing.

IP and the Postal Service

Imagine you send two letters, one to your neighbor and one across the country. From your perspective, all you have to do is send the letters and everything is done. From the post offices standpoint, they treat the two letters very differently. You can imagine the first letter being sent by a switch (locally) and the second must be processed by a router (postal routing center). TCP acts like the people sending and receiving the letters, while IP acts like the post office itself.
